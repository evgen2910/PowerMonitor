# ⚡ ESP32 Power Monitor

> Моніторинг мережевого живлення та заряду батареї резервного живлення з Telegram-сповіщеннями, веб-інтерфейсом та відстеженням відключень навіть після повного знеструмлення.

## Можливості

- **Миттєві Telegram-сповіщення** про зникнення та відновлення живлення
- **Тривалість відключення** — точний підрахунок часу, включно з випадками коли батарея сіла і плата перезавантажилась
- **Постійне збереження подій** в NVS flash — момент зникнення зберігається навіть при повному знеструмленні
- **Моніторинг батареї** — напруга і відсоток заряду через дільник напруги на ADC
- **Веб-інтерфейс** — дашборд реального часу, налаштування, журнал, OTA-оновлення
- **WiFi Captive Portal** — якщо не підключається, відкриває точку доступу для налаштування
- **Два WiFi-профілі** — основна і резервна мережа
- **OTA оновлення** через веб-браузер (drag & drop `.bin`)
- **mDNS** — доступ за іменем `http://powermon.local`
- **Черга повідомлень** — 20 слотів з автоматичним повтором при невдачі

## Схема підключення

### Датчик живлення (цифровий вхід)

Підключіть сигнал наявності мережі (наприклад, вихід реле або оптопари від блока живлення) до цифрового GPIO.

```
Блок живлення 5В
    │
   [R 10kΩ]      ← підтяжка до 3.3В (або використовуй INPUT_PULLUP)
    │
    ├──────────── GPIO (цифровий вхід)
    │
   GND
```

**Рекомендовані GPIO:**

| Плата | GPIO для живлення | GPIO для ADC батареї |
|---|---|---|
| ESP32-C3 SuperMini | GPIO 3 | GPIO 4 |
| ESP32 classic | GPIO 34 (input-only) | GPIO 36 (VP) |
| ESP32-S2 / S3 | GPIO 5 | GPIO 6 |

### Вимірювання напруги батареї (дільник напруги)

```
V_batt ──[R1]──┬──[R2]── GND
               │
             GPIO (ADC)
```

Напруга на GPIO = V_batt × R2 / (R1 + R2) — має бути **не більше 3.3В**.

**Рекомендовані номінали резисторів:**

| Тип батареї | Напруга | R1 | R2 | V_pin @ max |
|---|---|---|---|---|
| 1S Li-ion | 3.0–4.2В | 10 кОм | 10 кОм | 2.1В ✅ |
| 2S Li-ion | 6.0–8.4В | 56 кОм | 27 кОм | 2.7В ✅ |
| **3S Li-ion** ⭐ | 9.0–12.6В | 100 кОм | 27 кОм | 2.6В ✅ |
| 4S Li-ion | 12.0–16.8В | 150 кОм | 30 кОм | 2.8В ✅ |

> ⚠️ **ADC ESP32-C3:** `analogReadMilliVolts()` може завищувати показання через нелінійність АЦП. Перевір реальну напругу мультиметром і скорегуй значення `batt_full_v` / `batt_empty_v` у налаштуваннях.

## Встановлення

### 1. Залежності

Тільки **ESP32 Arduino Core** (≥ 2.0.0) — жодних сторонніх бібліотек.

В Arduino IDE: `Tools → Board → Boards Manager → ESP32 by Espressif Systems`

### 2. Вибір плати

Відкрий `Config.h` і розкоментуй рядок для своєї плати:

```cpp
// #define BOARD_ESP32C3   // ← для ESP32-C3 SuperMini / XIAO
// #define BOARD_ESP32S2   // ← для ESP32-S2
// #define BOARD_ESP32S3   // ← для ESP32-S3
// нічого не розкоментовано → ESP32 classic
```

### 3. Налаштування Arduino IDE для ESP32-C3 SuperMini

| Параметр | Значення |
|---|---|
| Board | ESP32C3 Dev Module |
| USB CDC On Boot | **Enabled** |
| CPU Frequency | 160MHz |
| Flash Size | 4MB |
| Partition Scheme | Default 4MB with spiffs |
| Upload Speed | 921600 |

### 4. Прошивка

`Sketch → Upload` або через OTA після першого завантаження.

## Перший запуск

1. При першому запуску (або якщо WiFi не налаштований) ESP32 відкриває точку доступу:
   - **SSID:** `PowerMon-Setup`
   - **Пароль:** `12345678`

2. Підключись до неї та перейди на `http://192.168.4.1`

3. Введи параметри WiFi та Telegram

4. Натисни **Зберегти і перезавантажити**

## Веб-інтерфейс

Після підключення до WiFi відкрий `http://powermon.local` або IP-адресу пристрою.

### Сторінки

| Сторінка | URL | Опис |
|---|---|---|
| Дашборд | `/` | Поточний стан, остання активність, кнопки дій |
| Налаштування | `/config` | Всі параметри, WiFi скан, GPIO |
| Журнал | `/logs` | Хронологія подій (60 записів) |
| OTA | `/ota` | Оновлення прошивки через браузер |

### Дашборд

- Живлення: ✅ ПРИСУТНЄ / ❌ ВІДСУТНЄ
- Тривалість відсутності (лічильник у реальному часі)
- WiFi: SSID, IP, рівень сигналу (dBm)
- NTP синхронізація, uptime
- Черга Telegram (відправлено / доставлено)
- **Батарея** (якщо увімкнено): напруга В, заряд %, прогрес-бар

## Налаштування Telegram

### 1. Створення бота

1. Напиши [@BotFather](https://t.me/BotFather) → `/newbot`
2. Скопіюй **Bot Token** у налаштування

### 2. Отримання Chat ID

**Автоматично:**
- Надішли будь-яке повідомлення боту
- У налаштуваннях натисни **🔍 Chat ID авто** — ID визначиться і збережеться

**Вручну:**
- Відкрий `https://api.telegram.org/bot<TOKEN>/getUpdates`
- Знайди поле `"id"` в об'єкті `"chat"`

### 3. Типи чатів

| Тип | Chat ID | Коли використовувати |
|---|---|---|
| Особистий | Позитивне число | Сповіщення тільки тобі |
| Група / канал | Від'ємне число (напр. `-1001234567`) | Сповіщення в групу |

## Відстеження відключень при знеструмленні

Це ключова особливість прошивки. Алгоритм:

```
Живлення зникло
    │
    ├─ Записує unix timestamp в NVS flash (namespace "pmevt")
    │  (NVS зберігається навіть при повному знеструмленні)
    │
    ├─ Надсилає Telegram "❌ ЗНИКЛО"
    │
    └─ [якщо батарея сіла → плата вимкнулась]

Живлення відновилось → плата завантажилась
    │
    ├─ Читає збережений timestamp з NVS
    ├─ Розраховує тривалість = now - saved_ts
    ├─ Надсилає Telegram "✅ ВІДНОВЛЕНО + тривалість"
    └─ Очищає NVS
```

> Якщо NTP не був синхронізований в момент зникнення, час зникнення позначається як "час невідомий", але подія все одно фіксується.

```

├── power_monitor.ino   — головний файл, setup() і loop()
```

## Параметри конфігурації

Всі параметри зберігаються в NVS і налаштовуються через веб-інтерфейс.

| Параметр | За замовч. | Опис |
|---|---|---|
| WiFi SSID 1/2 | — | Основна та резервна мережа |
| Bot Token | — | Telegram Bot API token |
| Chat ID | — | ID особистого чату |
| Group Chat ID | — | ID групи або каналу |
| Device Name | `Power Monitor` | Назва у повідомленнях |
| mDNS Name | `powermon` | Ім'я для `.local` |
| NTP Server | `pool.ntp.org` | Сервер часу |
| Timezone | `+2:00` (7200 сек) | Зміщення від UTC |
| Power Pin | плата-залежно | GPIO датчика живлення |
| Pin Mode | INPUT | INPUT / PULLUP / PULLDOWN |
| Active Low | `false` | Інвертувати логіку сигналу |
| Battery Enable | `false` | Увімкнути моніторинг АЦП |
| Battery Pin | плата-залежно | GPIO для ADC |
| R1 / R2 | 100 / 27 кОм | Дільник напруги (3S за замовч.) |
| Battery Cells | 3S | 1S–4S (визначає full/empty автоматично) |
| Notify: power on | ✅ | Сповіщення при відновленні |
| Notify: power off | ✅ | Сповіщення при зникненні |
| Notify: boot | ✅ | Сповіщення при старті |
| Notify: batt low | ✅ | Сповіщення при заряді ≤ 20% |
